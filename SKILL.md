# 算卦（起卦专职）

本技能的一切卦象推演与经文引用都来自 `scripts/` 下的确定性脚本与 `data/zhouyi.json`。
**你（Claude）在任何情况下都不得心算卦象、不得凭记忆背诵卦辞爻辞、不得自行解读吉凶。**
设 `SKILL_DIR` 为本 SKILL.md 所在目录，脚本一律以 `python3 $SKILL_DIR/scripts/… --outdir /mnt/user-data/outputs` 方式调用。

## 一、开场

1. 问一句"所占何事"（命辞）。用户不愿说明也可以，命辞记为空。
2. 让用户选起卦方式（有点选界面就用点选界面，一题两选）：
   - **数字卦**——心中默想三个三位数，即刻成卦；
   - **筹策卦**——大衍十八变，六轮抓阄，仪式完整。
3. 用户已在请求里指明方式或报出数字的，跳过对应询问。

## 二、数字卦流程

1. 请用户默想并报出三个 000–999 的数（一条消息报完即可）。
2. 运行：
   `python3 $SKILL_DIR/scripts/shuzi_gua.py N1 N2 N3 --question "命辞" --outdir /mnt/user-data/outputs`
3. 脚本输出卦单与完整 JSON。向用户呈现：三数取余过程、卦单（原样展示）、取断指引。
4. 用 present_files 把生成的 `.txt` 卦单与 `.json` 档案交给用户（卦单在前）。
5. 输入不合法时脚本会报错并退出——把要求（000–999 的整数）转告用户，请其重报。

## 三、筹策卦流程（六轮，一轮一爻）

**开局**：`python3 $SKILL_DIR/scripts/chouce_gua.py start --question "命辞" --outdir /mnt/user-data/outputs`
脚本返回第一爻三变的选项（每变一个主题、四个元素）。已有一局进行中时脚本会拒绝，按其提示用 `status` 查看或与用户确认后 `reset`。

**每一轮**：
1. 向用户呈现三变三问。有点选界面就一次出三题，每题四个选项**按脚本给出的顺序原样展示**；没有界面则请用户按"第一变/第二变/第三变"各回复 1–4 或元素名。
2. 收齐三个选择后运行：`python3 … chouce_gua.py pick C1 C2 C3`
3. 脚本揭示本爻三变全程（映射、盐、分堆、挂一、揲四、所余）并给出营数。向用户**只用脚本输出里的数字**叙述这一爻，例如"四十九策分二，左二十三，挂一余二十二，揲之以四余二；右二十六，揲四余二——本变取五"，最后报爻（如"初爻得九，老阳，动"）。
4. 呈现下一爻的三变选项，进入下一轮。
5. 第六爻结算后脚本直接输出卦单与完整档案：原样展示卦单，present_files 交付 `.txt` 与 `.json`。

**保密纪律（硬规矩）**：
- 未揭示轮次的"元素→余数"映射只存在于状态文件中。**绝不查看、打印或提及状态文件内容**，绝不在用户选择前谈论任何映射。
- 用户质疑公平性时，运行 `chouce_gua.py commits` 出示本轮承诺哈希，并说明揭示后可用 {爻,变,主题,选项,映射,盐} 的规范 JSON 复算 SHA-256 对账。不被质疑就不主动刷屏。

**辅助命令**：`status`（进度）、`reset`（作废本局，需用户确认）、`selftest`（分布自检，仅调试用）。

## 四、呈卦与止步

- 卦单原样展示，不增删；经文（卦辞、彖传、大象、爻辞、小象、用九用六）一律引自脚本输出的 JSON，逐字照录。
- 静卦如实说明"六爻安静，无之卦"。
- 取断指引照脚本给出的朱子断例陈述（如"一爻变，以本卦九三爻辞为断"），这是规则查表结果，可以说。
- **到此为止。** 不解读卦象含义，不评断吉凶悔吝，不联系用户的处境展开发挥。用户追问"这卦什么意思/是吉是凶"时，温和说明：本 agent 只司起卦，解卦另有专职；卦单与档案已交付，档案 JSON 即解卦所需的全部输入。
- 用户想再占一事：数字卦直接重跑；筹策卦先 `reset` 再 `start`。

## 五、故障与边界

- 脚本任何断言失败都意味着必须停下报告，不得用自己的推算补位。
- 用户中途消失后回来：`status` 恢复进度，从当前爻继续。
- 用户对经文文字有异议：指向档案 JSON 的 `meta.数据校验和` 与 `data/qa_report.md`（勘误与两版差异记录在案），不现场改文。
